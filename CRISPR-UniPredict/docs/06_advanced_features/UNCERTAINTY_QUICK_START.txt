OVERVIEW
========

Estimate prediction confidence with uncertainty intervals.
Quantify reliability of predictions.

QUICK START (5 minutes)
======================

# Load model and create estimator
from utils.uncertainty import UncertaintyEstimator

estimator.calibrate({
    'predictions': val_predictions,

    sgrna_onehot, sgrna_label,
    task_type='on_target'

print(f"Prediction: {result.prediction:.4f}")
print(f"95% CI: [{result.confidence_interval[0]:.4f}, {result.confidence_interval[1]:.4f}]")
print(f"Reliable: {result.is_reliable}")

UNCERTAINTY METHODS
===================

1. Monte Carlo Dropout
   - Run N forward passes with dropout enabled
   - Compute mean and std of predictions
   - Std = uncertainty
   - Best for: Accuracy
   - Time: 100x slower

2. Ensemble Variance
   - Use variance across ensemble members
   - Higher variance = higher uncertainty
   - Best for: Speed + accuracy
   - Time: 5x slower (5 models)

3. Conformal Prediction
   - Calibrate on validation set
   - Guarantees coverage at specified level
   - Best for: Guarantees
   - Time: 1x (same as prediction)

4. Temperature Scaling
   - Learn temperature parameter
   - Calibrate probabilities
   - Best for: Speed
   - Time: 1x (same as prediction)

RESULT FIELDS
=============

UncertaintyResult contains:
  - prediction: Point estimate
  - uncertainty: Uncertainty measure
  - confidence_interval: (lower, upper)
  - method: Estimation method
  - is_reliable: Reliability flag
  - reliability_score: Score 0-1

Access as:
  result.prediction
  result.uncertainty
  result.confidence_interval
  result.is_reliable
  result.reliability_score

==============

TROUBLESHOOTING
===============

"Calibration not performed"
  → Call estimator.calibrate() first

"Coverage too low"
  → Use lower confidence level
  → Check calibration data
  → Use conformal prediction

"Intervals too wide"
  → Use ensemble for better estimates
  → Increase calibration data
  → Check data quality

"CUDA out of memory"
  → Reduce n_samples for MC dropout
  → Use CPU: device='cpu'
  → Use ensemble instead
