================================================================================
WHY SINGLE MODEL FAILS ON ON-TARGET
================================================================================

ROOT CAUSE 1: SEVERE CLASS IMBALANCE
  - On-target samples: 291,639 (8.57%)
  - Off-target samples: 3,110,438 (91.43%)
  - Ratio: 10.7:1 (off-target dominates)
  - Impact: Model learns off-target patterns better

ROOT CAUSE 2: TASK DIFFICULTY MISMATCH
  - Off-target: Binary classification (easy)
    - Only 2 possible outputs
    - Clear decision boundary
    - Large dataset (2.5M samples)
  - On-target: Continuous regression (hard)
    - Infinite possible values (0.0-1.0)
    - Requires precise prediction
    - Small dataset (291K samples)
    - Noisy labels

ROOT CAUSE 3: MULTI-TASK LEARNING TRADE-OFF
  - Model must balance both tasks with single architecture
  - Shared representations not optimal for either task
  - Gradient flow dominated by off-target (easier, more samples)
  - On-target receives insufficient learning signal

ROOT CAUSE 4: LOSS WEIGHTING INSUFFICIENT
  - Increased on-target weight from 1.0 to 5.0 (5×)
  - Still insufficient to overcome sample count imbalance
  - Off-target: 2.5M samples × 0.5 weight = 1.25M gradient units
  - On-target: 0.3M samples × 5.0 weight = 1.5M gradient units
  - Fundamental difficulty of regression task remains

================================================================================
WHY ENSEMBLE WILL SOLVE THIS
================================================================================

SOLUTION 1: MODEL DIVERSITY
  - Train 5 models with different configurations
  - Different random seeds → Different initialization
  - Different loss weights → Different specializations
  - Different architectures → Different representations
  - Diversity reduces correlation between errors

SOLUTION 2: TASK-SPECIFIC OPTIMIZATION
  - Variant 0: Default (balanced)
  - Variant 1: Different seed (initialization diversity)
  - Variant 2: Off-target optimized (weight 0.5)
  - Variant 3: On-target optimized (weight 3.0)
  - Variant 4: Alternative architecture
  - Each variant specializes for different aspects

SOLUTION 3: META-MODEL LEARNING
  - Train meta-model on validation predictions
  - Input: Predictions from 5 base models (10 dimensions)
  - Output: Final on-target + off-target predictions
  - Meta-model learns optimal combination
  - Better than simple averaging

SOLUTION 4: REDUCED VARIANCE
  - Multiple models average out individual mistakes
  - Ensemble prediction more stable than single model
  - Better generalization to unseen data
  - More robust to outliers and noise
